üéí What is a Variable?

The invisible box that stores your data ‚Äî let's explore how it works!

üì¶ The Invisible Box Analogy

Imagine you have an invisible box that can store information. You can place anything you need to remember inside the box. When you need it, you simply find it by the box's label.


In programming, this invisible box is called a variable ‚Äî a memory location that can hold one piece of data at any given time.

üí°
CAPS DefinitionA variable refers to a memory location that can store data of a particular data type. Each variable has a unique name and can be used to store data.

‚≠ê 5 Key Properties of Variables
1
Stores one piece of data at a time

Like a box, only one thing fits at a time.

2
Always stays in the same memory location

You always know exactly where to find it.

3
The value can change

You can replace the old data with new data at any time.

4

A program can have hundreds of variables, each with a unique name.

5
Values only exist while the program runs

When the program closes, all variable values disappear from memory.

üè∑Ô∏è Naming Conventions

Good names make your code easy to understand ‚Äî there are strict rules AND best practice guidelines.

üö´ Naming RULES (Compulsory ‚Äî will cause errors if broken)
1
Names must be unique

No two variables or components can have the same name.

2
No spaces allowed

Use CamelCase instead: rAmountPaid not r Amount Paid

3
Cannot start with a number

2count ‚ùå but count2 ‚úÖ

4
No special characters

Except underscore ( _ ). No !, @, #, $, %, etc.

‚úÖ Naming CONVENTIONS (Best Practice)
1
Start with a type prefix letter

s for String, i for Integer, r for Real, b for Boolean, c for Char

2
Use CamelCase

First prefix lowercase, each new word starts with uppercase: rAmountPaid

3
Be descriptive

iAge not just i ‚Äî names should tell you what data they hold

üìù Declaring Variables & Components

Before you can use a variable, you must declare it ‚Äî tell Delphi to reserve memory for it.

üìê Declaration Syntax

Variables are declared in the var section, using a colon to separate the name from the type, ending with a semicolon:

Syntax var // Start of variable declarations
  VariableName : DataType; // Declare one variable
  Name1, Name2 : DataType; // Declare multiple of the same type

Rules to remember:

‚Ä¢

The var keyword starts the section

‚Ä¢

Colon : separates name from type

‚Ä¢

Semicolon ; ends each declaration

‚Ä¢

Indent declarations for readability

‚Ä¢

Multiple same-type vars on one line, separated by commas

Example var
  rTotal : Real;
  iMinimum, iMaximum : Integer;
  sName : String;
  cGender : Char;
  bValid : Boolean;

‚úèÔ∏è Assigning Values to Variables

Putting data INTO a variable using the assignment operator :=

‚ö° The Assignment Operator ‚Äî :=

The := (colon-equals) operator stores a value in a variable. Think of it as an arrow pointing LEFT ‚Äî "put this value into this variable."


variableName := value or expression
üö´
Common Mistake!Using = instead of := will cause a syntax error! In Pascal/Delphi, = is only used for comparison.
Examples var
  iAge : Integer;
  sName : String;
  rHeight : Real;
  bPassed : Boolean;
  cGrade : Char;
begin
  iAge := 16;
  sName := 'Thabo';
  rHeight := 1.75;
  bPassed := True;

end;

üì• Reading Input FROM Components

When a user types in a TEdit component, you read it using the .Text property. It always comes as a String!

Reading from TEdit // Reading text ‚Äî stays as String
sName := edtName.Text;

// Reading a number ‚Äî MUST cast (convert) first!
iMark := StrToInt(edtMark.Text);
rPrice := StrToFloat(edtPrice.Text);
Writing output TO components // Output to TLabel ‚Äî must convert number to String
lblResult.Caption := 'Hello, ' + sName;
lblAge.Caption := IntToStr(iAge);
lblPrice.Caption := FloatToStr(rPrice);
üî¢
TSpinEdit ‚Äî Value propertyThe TSpinEdit uses .Value (not .Text), which gives you an Integer directly ‚Äî no casting needed!

ü§î Why Do We Need to Convert Types?

Every TEdit component stores its data as a String. You cannot do maths on a String. The solution is to convert the String to an Integer or Real first, do your calculation, then convert the result back to String to display it.

üêõ Types of Programming Errors

Three types of errors ‚Äî learn to identify, explain and fix each one!

üî¥ Syntax Error(Compile-time)

Code that breaks the grammar rules of the programming language. Delphi catches these before the program runs and shows them in the Messages panel.

// Missing semicolon:
iAge := 16 ‚Üê ERROR!

// Wrong assignment operator:
sName = 'Thabo'; ‚Üê ERROR!

// Undeclared variable:
iScore := 0; ‚Üê ERROR if not declared!
üü° Runtime Error(While running)

Code that compiles fine but crashes while running because it tries to do something impossible under certain conditions.

// User types 'abc' in a mark field:
iMark := StrToInt(edtMark.Text);
‚Üê CRASH! 'abc' ‚â† number

// Division by zero:
rResult := rTotal / rDivisor;
‚Üê CRASH if rDivisor = 0!
üü¢ Logic Error(Hardest to find!)

Code that compiles AND runs, but produces wrong results because the algorithm is incorrect.

// Wrong average calculation:
rAverage := iMark1 + iMark2 / 2;
‚Üê Should be: (iMark1+iMark2)/2

// Wrong swap (value lost):
sName := sSurname; // ‚Üê sName's original
sSurname := sName; ‚Üê value is gone!
